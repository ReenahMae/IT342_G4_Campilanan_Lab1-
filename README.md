Project Description

MiniAuth is a full-stack authentication system that allows users to:

Register with first name, middle name, last name, email, and password

Log in using JWT authentication

Access a protected dashboard

Update profile settings

Log out securely

Protected pages such as Dashboard and Profile cannot be accessed without authentication.

How to Run Backend

Start XAMPP.

Start Apache

Start MySQL

Create database in phpMyAdmin:

Open http://localhost/phpmyadmin

Create a database named:

miniauth_db


Configure application.properties
File location:

backend/miniauth/src/main/resources/application.properties

spring.datasource.url=jdbc:mysql://localhost:3306/miniauth_db?useSSL=false&serverTimezone=UTC
spring.datasource.username=root
spring.datasource.password=

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

server.port=8080


Run the backend:

From IntelliJ:

Run MiniauthApplication.java

Or from terminal inside backend/miniauth:

mvn clean install
mvn spring-boot:run


Backend runs at:

http://localhost:8080

How to Run Web

Open terminal inside the web folder:

cd web


Install dependencies:

npm install


Start the React app:

npm start


Web runs at:

http://localhost:3000


Make sure the backend is running before logging in.

How to Run Mobile

Mobile version is not yet implemented.

The mobile application will use the same backend at:

http://localhost:8080

API Endpoints
Register

POST

/api/auth/register


Request body:

{
  "firstName": "Mae",
  "middleName": "",
  "lastName": "Campilanan",
  "email": "mae@test.com",
  "password": "123456"
}


Response:

{
  "message": "Registered successfully"
}

Login

POST

/api/auth/login


Request body:

{
  "email": "mae@test.com",
  "password": "123456"
}


Response:

{
  "token": "JWT_TOKEN",
  "user": {
    "id": 1,
    "firstName": "Mae",
    "middleName": "",
    "lastName": "Campilanan",
    "email": "mae@test.com"
  }
}

Get Current User

GET

/api/users/me


Header:

Authorization: Bearer <token>


Returns the logged-in user's details.

Update Profile

PUT

/api/users/update


Header:

Authorization: Bearer <token>


Allows updating:

First name

Middle name

Last name

Password (optional)
